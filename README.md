# Dekor auto posting (Telegram)

Бот публикует посты из Excel по очереди **раз в 2 дня** и после окончания возвращается к первому (вечный цикл).  
Прогресс хранится в `state.json`, поэтому при перезапуске продолжает с нужного места.

## Что нужно

- Python 3.10+ (у вас уже стоит Python 3.12)
- Файл `posts.xlsx` в корне проекта (у вас он есть)
- Лист с постами: `Posts` (в вашем файле так и называется; скрипт ищет также `posts` без учета регистра)

## Формат Excel (лист `Posts`)

Ожидаемые колонки:

- `ID` — идентификатор поста (используется вместе с листом `Queue`)
- `text` — текст поста (можно HTML: `<b>...</b>`, как у вас)
- `Photo1`..`Photo10` — ссылки на фото (URL) или локальные пути к файлам
- `Video1`..`Video10` — ссылки на видео (URL) или локальные пути к файлам

Публикация:

- если есть медиа — отправляется медиа (по возможности альбомом), а подпись/текст прикрепляется к первому элементу
- если медиа нет — отправляется текстом

## Настройка

1) Установите зависимости:

```bash
python -m pip install -r requirements.txt
```

2) Создайте `.env` рядом с `.env.example`:

- `TELEGRAM_BOT_TOKEN` — токен бота
- `TELEGRAM_CHAT_ID` — куда постить

Пример:

```env
TELEGRAM_BOT_TOKEN=123:ABC...
TELEGRAM_CHAT_ID=-1001234567890
```

### Если в Excel заполнены `Settings/State/Queue`

В вашем `posts.xlsx` уже есть листы:

- `Settings`: `chat_id`
- `State`: `Postindex` (какой шаг следующий)
- `Queue`: `Step` + `PostID` (порядок публикации)

В этом режиме можно **не задавать** `TELEGRAM_CHAT_ID` в `.env` — скрипт возьмёт `chat_id` из `Settings`.  
Также скрипт будет **обновлять `State/Postindex`** после каждой публикации.  
Маркер `PostID=recycle` в конце очереди воспринимается как “вернуться к началу”.

### Частота публикации через Excel (лист `Frequency`)

Можно добавить лист `Frequency` с одной строкой `frequency`:

- `1` = каждый день
- `2` = через день (раз в 2 дня)
- `3` = через 2 дня (раз в 3 дня)

Если лист `Frequency` есть и в нём задано значение, оно **перекрывает** `INTERVAL_DAYS` из `.env`.

### Как узнать `TELEGRAM_CHAT_ID`

- **Для группы/лички**: напишите боту любое сообщение, затем запустите команду:

```bash
python -m src.dekor_bot.print_chat_id
```

и посмотрите ID в консоли.

- **Для канала**: добавьте бота админом канала (право “Публикация сообщений”), затем перешлите сообщение из канала боту или используйте сторонний способ получения `-100...` (часто проще всего через `@userinfobot`/аналог).

## Запуск

```bash
python -m src.dekor_bot.main
```

По умолчанию первый пост отправится сразу (`START_IMMEDIATELY=true`), следующий — через 2 дня.

### Принудительно отправить следующий пост (игнорируя таймер)

В любой момент можно насильно “прокрутить” очередь и отправить следующий пост:

```bash
python -m src.dekor_bot.force_next
```

Скрипт возьмёт следующий пост по `State/Queue`, отправит его сразу и обновит состояние так, будто время подошло сейчас.

## Запуск через Планировщик задач Windows (рекомендовано)

Чтобы бот не висел постоянно запущенным, можно поставить `RUN_ONCE=true` и запускать задачу, например, **раз в день** (или раз в час).  
Скрипт сам решит: “пора постить или нет”, и завершится.
